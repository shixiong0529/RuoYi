**技术栈**: Spring Boot 4.0.3 + Apache Shiro + MyBatis + Thymeleaf

### 1.4 请求完整流程

```
HTTP Request
    │
    ├─[1] ShiroFilterFactoryBean 过滤链
    │       ├─ LogoutFilter           /logout
    │       ├─ CaptchaValidateFilter  验证码校验
    │       ├─ CsrfValidateFilter     CSRF token 校验
    │       ├─ KickoutSessionFilter   多会话踢出
    │       ├─ OnlineSessionFilter    在线用户更新
    │       └─ SyncOnlineSessionFilter 会话同步
    │
    ├─[2] UserRealm 认证/授权
    │       ├─ doGetAuthenticationInfo() 认证
    │       └─ doGetAuthorizationInfo()  授权
    │
    ├─[3] AOP 切面
    │       ├─ LogAspect              @Log 操作日志
    │       ├─ DataScopeAspect        @DataScope 数据权限

    │       └─ PermissionsAspect      @RequiresPermissions 权限
    │
    ├─[4] Controller → Service → Mapper → DB
    │
    └─[5] 异步处理（AsyncManager + AsyncFactory）
            └─ 10ms 延迟异步写入日志/会话数据
```

## 二、核心模块功能说明

### 2.1 ruoyi-admin — Web 入口模块

**职责**: 整个应用的启动入口，聚合所有子模块，提供 Web 层接口。

**关键路径**: `ruoyi-admin/src/main/java/com/ruoyi/`

```
RuoYiApplication.java              ← Spring Boot 启动类
web/controller/
├── common/
│   ├── CommonController.java       ← 文件上传/下载
│   └── SysCaptchaController.java  ← 验证码生成（math/char 两种类型）
├── system/                         ← 用户/角色/菜单/部门/岗位/字典管理
├── monitor/                        ← 在线用户/操作日志/登录日志/缓存管理
└── tool/                           ← 代码生成入口
```

**主要配置** (`application.yml`):
- 文件上传限制：单文件 10MB，总计 20MB
- XSS 防护：默认开启，排除 `/system/notice/*`
- CSRF 防护：默认关闭（`csrf.enabled: false`）

### 2.2 ruoyi-framework — 核心框架模块

**职责**: 提供安全认证、AOP 切面、异步管理等基础设施能力。

**关键路径**: `ruoyi-framework/src/main/java/com/ruoyi/framework/`

#### Shiro 安全体系

```
config/ShiroConfig.java             ← Shiro 全局配置（缓存/Session/过滤链）
shiro/
├── realm/UserRealm.java            ← 自定义 Realm（认证+授权核心）
├── service/
│   ├── SysLoginService.java        ← 登录业务逻辑（IP黑名单/密码重试/锁定）
│   ├── SysPasswordService.java     ← 密码比对服务（含错误次数计数）
│   └── SysShiroService.java        ← 会话数据库持久化
├── session/
│   ├── OnlineWebSessionManager.java ← 自定义 Session 管理器
│   ├── OnlineSessionDAO.java        ← Session 数据库访问
│   └── OnlineSessionFactory.java    ← Session 工厂（含反序列化）
└── web/filter/
    ├── captcha/CaptchaValidateFilter.java  ← 验证码过滤
    ├── csrf/CsrfValidateFilter.java        ← CSRF 过滤
    ├── kickout/KickoutSessionFilter.java   ← 多会话踢出
    ├── online/OnlineSessionFilter.java     ← 在线状态更新
    └── sync/SyncOnlineSessionFilter.java   ← 会话同步
```

#### AOP 切面

```
aspectj/
├── LogAspect.java          ← 拦截 @Log，记录操作日志（含参数/返回值/耗时）
├── DataScopeAspect.java    ← 拦截 @DataScope，动态拼接数据权限 SQL
├── DataSourceAspect.java   ← 拦截 @DataSource，切换 MyBatis 数据源
└── PermissionsAspect.java  ← 拦截 @RequiresPermissions，多角色权限匹配
```

#### 异步管理

```
manager/
├── AsyncManager.java             ← 单例异步管理器（ScheduledExecutorService）
├── ShutdownManager.java          ← 应用关闭时优雅停止线程池
└── factory/AsyncFactory.java     ← 异步任务工厂
    ├── recordOper()              ← 异步记录操作日志
    ├── recordLogininfor()        ← 异步记录登录日志
    └── syncSessionToDb()         ← 异步同步会话到数据库
```

## 三、关键代码路径标注

### 3.1 登录认证流程

```
前端 POST /login
    │
    ▼
[Filter] CaptchaValidateFilter
    └─ 比对 Session 中的验证码
    │  失败 → 返回错误
    ▼
[Filter] CsrfValidateFilter
    └─ 校验 CSRF token（若开启）
    ▼
[Shiro] UserRealm.doGetAuthenticationInfo()
    └─ 路径：ruoyi-framework/.../shiro/realm/UserRealm.java
    ▼
[Service] SysLoginService.login(username, password)
    └─ 路径：ruoyi-framework/.../shiro/service/SysLoginService.java
    ├─ IP 黑名单检查  → ISysConfigService.selectConfigByKey("sys.login.blackIPList")
    ├─ 用户查询       → ISysUserService.selectUserByLoginName(username)
    ├─ 密码验证       → SysPasswordService.validate(user, password)
    │                    路径：.../shiro/service/SysPasswordService.java
    │                    ├─ 错误次数 ≥ maxRetryCount → 抛 UserPasswordRetryLimitExceedException
    │                    └─ 密码正确 → 清除计数缓存
    └─ 异步记录登录日志 → AsyncManager.execute(AsyncFactory.recordLogininfor())
    ▼
[Shiro] Session 创建
    └─ OnlineWebSessionManager → OnlineSessionDAO（写库）
    ▼
登录成功 → 跳转 /index
```

**涉及关键文件**:
- `ruoyi-framework/src/main/java/com/ruoyi/framework/shiro/realm/UserRealm.java`
- `ruoyi-framework/src/main/java/com/ruoyi/framework/shiro/service/SysLoginService.java`
- `ruoyi-framework/src/main/java/com/ruoyi/framework/shiro/service/SysPasswordService.java`
- `ruoyi-framework/src/main/java/com/ruoyi/framework/shiro/session/OnlineWebSessionManager.java`

### 3.2 请求鉴权流程

```
已登录用户发起请求
    │
    ▼
[Shiro Filter] 路由规则匹配（ShiroConfig）
    ├─ /html/**、/css/**、/js/** → anon（放行）
    ├─ /login、/register       → captcha + anon
    └─ /system/**、/monitor/** → user + kickout + online + csrf
    ▼
[Shiro] UserRealm.doGetAuthorizationInfo()
    └─ 路径：ruoyi-framework/.../shiro/realm/UserRealm.java
    ├─ 管理员(admin) → 赋予 *:*:* 通配权限
    └─ 普通用户
        ├─ ISysRoleService.selectRoleKeys(userId)     ← 角色列表
        └─ ISysMenuService.selectPermsByUserId(userId) ← 权限字符串列表
    ▼
[AOP] PermissionsAspect 拦截 @RequiresPermissions
    └─ 将权限字符串写入 PermissionContextHolder（线程上下文）
    ▼
[AOP] DataScopeAspect 拦截 @DataScope
    └─ 根据用户角色的数据范围，动态拼接 SQL WHERE 条件
    ├─ DATA_SCOPE_ALL  (1) → 全部数据，不追加条件
    ├─ DATA_SCOPE_CUSTOM (2) → 自定义部门，拼接 dept_id IN (...)
    ├─ DATA_SCOPE_DEPT (3) → 本部门，拼接 dept_id = ?
    ├─ DATA_SCOPE_DEPT_AND_CHILD (4) → 本部门及子部门
    └─ DATA_SCOPE_SELF (5) → 仅本人，拼接 user_id = ?
```

**涉及关键文件**:
- `ruoyi-framework/src/main/java/com/ruoyi/framework/config/ShiroConfig.java`
- `ruoyi-framework/src/main/java/com/ruoyi/framework/aspectj/DataScopeAspect.java`
- `ruoyi-framework/src/main/java/com/ruoyi/framework/aspectj/PermissionsAspect.java`

### 3.3 操作日志记录流程

```
Controller 方法标注 @Log(title="用户管理", businessType=BusinessType.INSERT)
    │
    ▼
[AOP] LogAspect @Before
    └─ 路径：ruoyi-framework/.../aspectj/LogAspect.java
    ├─ 获取当前用户（ShiroUtils.getSysUser()）
    └─ 记录开始时间（ThreadLocal<Long>）
    ▼
方法执行
    ▼
[AOP] @AfterReturning / @AfterThrowing
    ├─ 成功 → status = SUCCESS，记录响应内容（isSaveResponseData=true 时）
    └─ 失败 → status = FAIL，记录异常信息
    ▼
handleLog() 组装 SysOperLog
    ├─ operIp（ShiroUtils.getIp()）
    ├─ operUrl（HttpServletRequest.getRequestURI()）
    ├─ operName（当前登录用户名）
    ├─ deptName（用户所属部门）
    ├─ method（被调用的类方法名）
    ├─ operParam（请求参数 JSON，排除 excludeParamNames 字段）
    ├─ jsonResult（返回结果 JSON，最多 2000 字符）
    └─ costTime（执行耗时毫秒）
    ▼
AsyncManager.execute(AsyncFactory.recordOper(operLog))
    └─ 延迟 10ms 异步写入 sys_oper_log 表
```

**涉及关键文件**:
- `ruoyi-framework/src/main/java/com/ruoyi/framework/aspectj/LogAspect.java`
- `ruoyi-framework/src/main/java/com/ruoyi/framework/manager/AsyncManager.java`
- `ruoyi-framework/src/main/java/com/ruoyi/framework/manager/factory/AsyncFactory.java`
- `ruoyi-common/src/main/java/com/ruoyi/common/annotation/Log.java`

### 3.5 定时任务执行流程

```
Quartz Scheduler（按 cronExpression 触发）
    │
    ▼
QuartzJobExecution.execute(JobExecutionContext)
    └─ 路径：ruoyi-quartz/.../util/QuartzJobExecution.java
    ▼
AbstractQuartzJob.execute()（模板方法）
    └─ 路径：ruoyi-quartz/.../util/AbstractQuartzJob.java
    ├─ before()   → 记录开始时间（ThreadLocal<Date>）
    ├─ doExecute()
    │   └─ JobInvokeUtil.invokeMethod(sysJob)
    │       └─ 路径：.../util/JobInvokeUtil.java
    │       ├─ 解析 invokeTarget 字符串
    │       │   格式：com.ruoyi.quartz.task.RyTask.ryParams('hello')
    │       ├─ ClassUtils.forName(className)   ← 加载类
    │       ├─ getMethod(methodName, types)    ← 获取方法
    │       └─ method.invoke(obj, args)        ← 反射调用
    └─ after()
        ├─ 计算耗时
        ├─ 构建 SysJobLog（status/exceptionInfo）
        └─ ISysJobLogService.addJobLog(sysJobLog) ← 写入执行日志
```

- `ruoyi-quartz/src/main/java/com/ruoyi/quartz/util/AbstractQuartzJob.java`

## 四、扩展点识别

### 4.1 可继承的接口/抽象类

| 扩展点 | 类型 | 路径 | 说明 |
|-------|------|------|------|
| 自定义 Shiro Realm | 抽象类 | `org.apache.shiro.realm.AuthorizingRealm` | 扩展认证/授权逻辑 |
| 自定义 Quartz 任务 | 接口 | `org.quartz.Job` | 通过 AbstractQuartzJob 扩展 |
| 自定义 Shiro 过滤器 | 抽象类 | `AccessControlFilter` | 新增过滤链节点 |
| 自定义异常 | 基类 | `com.ruoyi.common.exception.base.BaseException` | 扩展业务异常 |
| 自定义实体 | 基类 | `com.ruoyi.common.core.domain.BaseEntity` | 所有实体类继承 |
| 自定义树形实体 | 基类 | `com.ruoyi.common.core.domain.TreeEntity` | 树形结构实体继承 |

### 4.2 注解扩展点

| 注解 | 位置 | 作用 | 扩展方式 |
|------|------|------|---------|
| `@Log` | Controller 方法 | 自动记录操作日志 | 修改 `businessType`/`excludeParamNames` |
| `@DataScope` | Service/Mapper | 数据行级权限过滤 | 配置 `deptAlias`/`userAlias` |
| `@DataSource` | Service/Mapper | 多数据源动态切换 | 配置 `value` 指定数据源名 |
| `@Excel` | 实体字段 | Excel 导入导出 | 配置 `name`/`width`/`dictType` |
| `@RepeatSubmit` | Controller 方法 | 防止重复提交 | 配置 `interval`/`message` |
| `@Sensitive` | 实体字段 | 敏感数据脱敏输出 | 配置脱敏类型 |
| `@Anonymous` | Controller 方法 | 标记为匿名可访问 | 直接标注，无需配置 |

### 4.4 代码生成模板扩展

Velocity 模板位于 `ruoyi-generator/src/main/resources/templates/`，可自定义添加新模板：

```
templates/
├── Entity.vm          ← 实体类（可修改字段注解、lombok 等）
├── Service.vm         ← Service 接口
├── ServiceImpl.vm     ← Service 实现
├── Mapper.java.vm     ← Mapper 接口
├── Mapper.xml.vm      ← MyBatis XML（可修改 SQL 模板）
├── Controller.vm      ← Controller（可修改 API 风格）
└── 自定义.vm          ← 新增模板（如 DTO、VO、单元测试等）
```

**添加新模板步骤**:
1. 在 `templates/` 目录新建 `.vm` 文件
2. 在 `VelocityUtils.getTemplateList()` 中注册新模板
3. 在 `VelocityUtils.getFileName()` 中配置输出文件名规则

### 4.5 定时任务扩展

在任意模块中新建任务类，无需额外配置：

```java
// 示例：自定义定时任务
@Component
public class MyTask {
    // invokeTarget 配置为：com.ruoyi.MyTask.execute('param')

        // 业务逻辑
    }
}
```

### 4.6 数据权限扩展

在 `DataScopeAspect` 中扩展新的数据范围类型：

```java
// DataScopeAspect.java 中新增数据范围类型
public static final String DATA_SCOPE_CUSTOM = "自定义";  // 已有
// 可新增如：
public static final String DATA_SCOPE_COMPANY = "公司级别";
```

### 4.7 异步任务扩展

在 `AsyncFactory` 中新增异步任务工厂方法：

```java
// AsyncFactory.java 新增任务
public static TimerTask myAsyncTask(Object param) {
    return new TimerTask() {
        @Override
        public void run() {
            // 异步业务逻辑
        }
    };
}
// 调用
AsyncManager.me().execute(AsyncFactory.myAsyncTask(param));
```

## 五、核心设计模式总结

| 模式 | 应用场景 | 实现位置 |
|------|--------|---------|
| **单例模式** | 异步管理器 | `AsyncManager.me()` |
| **模板方法** | 定时任务执行 | `AbstractQuartzJob` |
| **工厂方法** | 异步任务创建 | `AsyncFactory` |
| **策略模式** | 数据权限范围 | `DataScopeAspect` 中的 5 种范围 |
| **AOP 代理** | 日志/权限/数据源 | `LogAspect`/`DataScopeAspect` 等 |
| **责任链** | Shiro 过滤链 | `ShiroFilterFactoryBean` |
| **观察者** | Session 事件 | Shiro Session 监听器 |
| **建造者** | 响应体构建 | `AjaxResult.success()/error()` |
