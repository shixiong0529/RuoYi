## 详细漏洞分析

### 1. SQL 注入（CRITICAL）

**问题描述**: 多个 MyBatis Mapper XML 文件使用 `${}` 而非 `#{}` 进行 SQL 参数拼接，存在 SQL 注入风险。

**受影响文件**:

- `ruoyi-generator/src/main/resources/mapper/generator/GenTableMapper.xml` 第169行
- `ruoyi-system/src/main/resources/mapper/system/SysUserMapper.xml` 第88、105、123行
- `ruoyi-system/src/main/resources/mapper/system/SysDeptMapper.xml` 第54行
- `ruoyi-system/src/main/resources/mapper/system/SysRoleMapper.xml` 第61行

**问题代码**:
```xml
<!-- GenTableMapper.xml 第169行 - 直接执行任意 SQL -->
<update id="createTable">
    ${sql}
</update>

<!-- SysUserMapper.xml - 数据范围拼接 -->
${params.dataScope}
```

**修复建议**:
- 使用 `#{}` 替代 `${}` 进行参数绑定
- 对 `dataScope` 参数在业务层进行严格白名单校验
- `GenTableMapper` 的 `createTable` 使用 `SqlUtil.filterKeyword()` 过滤 SQL 关键字

---

### 2. 不安全的反序列化（HIGH）

**问题描述**: `OnlineSessionFactory` 使用 `ObjectInputStream` 直接反序列化不可信数据，存在远程代码执行（RCE）风险。

**受影响文件**:
- `ruoyi-framework/src/main/java/com/ruoyi/framework/shiro/session/OnlineSessionFactory.java` 第32-50行

**问题代码**:
```java
ByteArrayInputStream bis = new ByteArrayInputStream(userOnline.getSessionData());
ObjectInputStream ois = new ObjectInputStream(bis);
OnlineSession onlineSession = (OnlineSession) ois.readObject();  // 危险！
```

**修复建议**:
- 使用 JSON 序列化替代 Java 原生序列化
- 或添加 `ObjectInputFilter` 白名单过滤允许反序列化的类
- 验证序列化数据的签名和完整性

---

### 3. 硬编码凭证（HIGH）

**问题描述**: 配置文件中存在明文数据库密码及 Druid 控制台弱密码。

**受影响文件**:
- `ruoyi-admin/src/main/resources/application-druid.yml` 第10-11行、第50-51行

**问题代码**:
```yaml
master:
  url: jdbc:mysql://localhost:3306/ry?...
  username: root
  password: password        # 明文密码

# Druid 控制台默认凭证
login-username: ruoyi
login-password: 123456      # 弱密码
```

**修复建议**:
- 使用环境变量或密钥管理服务（如 HashiCorp Vault）存储敏感信息
- 修改 Druid 控制台默认凭证为强密码
- 限制 Druid 监控页面的访问 IP 白名单

---

### 4. 路径遍历（MEDIUM-HIGH）

**问题描述**: 文件下载功能路径验证仅检查 `..`，防护不完整，可能被绕过导致任意文件读取。

**受影响文件**:
- `ruoyi-admin/src/main/java/com/ruoyi/web/controller/common/CommonController.java` 第140-157行

**问题代码**:
```java
@GetMapping("/download/resource")
public void resourceDownload(String resource, ...) throws Exception {
    if (!FileUtils.checkAllowDownload(resource)) {
        throw new Exception(...);
    }
    String downloadPath = localPath + FileUtils.stripPrefix(resource);  // 路径拼接
    FileUtils.writeBytes(downloadPath, response.getOutputStream());
}

// checkAllowDownload 仅检查 ..
public static boolean checkAllowDownload(String resource) {
    if (StringUtils.contains(resource, "..")) {  // 只检查 ..，不够充分
        return false;
    }
    ...
}
```

**修复建议**:
- 使用 `Path.normalize()` 规范化路径
- 验证最终解析的文件路径是否在预期目录内
- 添加符号链接检查

---

### 5. 敏感信息泄露（MEDIUM）

**问题描述**: 全局异常处理将异常消息直接返回给客户端，暴露内部实现细节。

**受影响文件**:
- `ruoyi-framework/src/main/java/com/ruoyi/framework/web/exception/GlobalExceptionHandler.java` 第70-82行
- `ruoyi-common/src/main/java/com/ruoyi/common/utils/ExceptionUtil.java` 第17-22行

**问题代码**:
```java
@ExceptionHandler(RuntimeException.class)
public AjaxResult handleRuntimeException(RuntimeException e, ...) {
    log.error("请求地址'{}',发生未知异常.", requestURI, e);
    return AjaxResult.error(e.getMessage());  // 直接将异常信息返回客户端
}
```

**修复建议**:
- 仅在日志中记录完整堆栈，客户端只返回通用错误消息
- 实现错误码映射机制，避免技术细节暴露
- 生产环境关闭 DEBUG 日志级别

---

### 6. XSS 防护不足（MEDIUM）

**问题描述**: XSS 过滤器对 GET 和 DELETE 请求完全跳过过滤，存在 XSS 攻击风险。

**受影响文件**:
- `ruoyi-common/src/main/java/com/ruoyi/common/xss/XssFilter.java` 第57-67行

**问题代码**:
```java
if (method == null || method.matches("GET") || method.matches("DELETE")) {
    return true;  // GET/DELETE 请求完全跳过 XSS 过滤
}
```

**修复建议**:
- 对所有 HTTP 方法启用 XSS 过滤
- 在响应端使用模板引擎自动转义
- 添加 `Content-Security-Policy` 响应头

---

### 7. CSRF 防护禁用（MEDIUM）

**问题描述**: CSRF 防护功能默认关闭，攻击者可诱导用户执行未授权操作。

**受影响文件**:
- `ruoyi-admin/src/main/resources/application.yml` 第150-151行

**问题代码**:
```yaml
csrf:
  enabled: false  # CSRF 防护已关闭
```

**修复建议**:
- 将 `enabled` 改为 `true` 启用 CSRF 防护
- 为 Cookie 设置 `SameSite=Strict` 属性
- 确保所有状态变更操作都验证 CSRF token

---

### 8. 权限验证缺失（MEDIUM）

**问题描述**: 文件上传/下载接口无权限注解，任意已登录用户均可访问。

**受影响文件**:
- `ruoyi-admin/src/main/java/com/ruoyi/web/controller/common/CommonController.java` 第46-97行

**问题代码**:
```java
@GetMapping("/download")
public void fileDownload(String fileName, Boolean delete, ...) {
    // 缺少 @RequiresPermissions 注解
}

@PostMapping("/upload")
public AjaxResult uploadFile(MultipartFile file) throws Exception {
    // 缺少权限验证
}
```

**修复建议**:
- 添加 `@RequiresPermissions` 注解
- 限制上传文件大小和类型
- 记录所有文件操作审计日志

---

### 9. 日志注入（LOW-MEDIUM）

**问题描述**: 用户输入未经清理直接写入日志，可能导致日志伪造。

**受影响文件**:
- `ruoyi-framework/src/main/java/com/ruoyi/framework/aspectj/LogAspect.java` 第112、178行

**修复建议**:
- 对日志内容移除换行符（`\n`、`\r`）和特殊字符
- 对敏感参数（密码、token）进行脱敏处理
- 使用参数化日志避免字符串拼接

---

## 优先级修复路线

### P0 - 立即修复
1. **SQL 注入** - `GenTableMapper.xml` 的 `${sql}` 可直接执行任意 SQL
2. **反序列化 RCE** - `OnlineSessionFactory.java` 存在远程代码执行风险
3. **硬编码凭证** - 移除明文密码，修改默认弱密码

### P1 - 高优先级
4. **路径遍历** - 完善文件下载路径验证逻辑
5. **敏感信息泄露** - 统一异常处理，不向客户端暴露技术细节
6. **CSRF** - 启用 CSRF 防护

### P2 - 中优先级
7. **XSS 过滤** - 对所有 HTTP 方法启用过滤
8. **权限验证** - 为文件操作接口添加权限控制
9. **日志注入** - 清理日志中的用户输入内容
